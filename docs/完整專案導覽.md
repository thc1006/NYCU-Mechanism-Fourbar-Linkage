## 🔧 常見問題 FAQ

### Q1: 我應該從哪份文檔開始？

**A**: 按順序：
1. `CLAUDE.md`（理解需求）
2. `解題流程與設計思考分析.md`（理解方法）
3. `步驟1`（開始計算）

### Q2: Python 代碼一定要執行嗎？

**A**: 強烈建議執行！

**理由**：
- 自動產生 5 個方案（手動設計很費時）
- 自動生成高品質圖表
- 展現你會使用優化工具
- 大幅增加報告的技術含量

**如果真的無法執行**：
- 可以使用文檔中的「典型結果」
- 但要註明是理論值

### Q3: Working Model 2D 一定要用嗎？

**A**: 是的，題目明確要求。

**替代方案**：
- 如果學校沒有授權，可以用試用版
- 或請同學協助
- 但一定要有模擬截圖

### Q4: 報告要多長？

**A**: 建議 15-25 頁（A4, 不含封面）

**頁數分配**：
- 第 1 部分：1 頁
- 第 2 部分：2-3 頁
- 第 3 部分：3-4 頁
- 第 4 部分：4-6 頁（含圖）
- 第 5 部分：4-6 頁（含圖表）
- 第 6 部分：2-3 頁

### Q5: LaTeX 公式在 HackMD 無法顯示？

**A**: 檢查以下：

1. `$` 符號緊鄰數學式（沒有空格）
   - ❌ `$ x = 1 $`
   - ✅ `$x = 1$`

2. 多行公式用 `$$` 包圍

3. 使用 `\begin{align}` 對齊等號

**所有文檔的公式都已經是正確格式！**

### Q6: 如果優化結果不符合預期？

**A**: 可能原因：

1. **約束太嚴格** → 放寬一些限制
2. **目標函數定義錯誤** → 檢查正負號
3. **演算法參數** → 增加迭代次數（`maxiter`）
4. **隨機性** → 多跑幾次取最佳

**解決方法**：
- 參考「實際執行」文檔的第 6.2 節
- 或使用文檔提供的典型結果
